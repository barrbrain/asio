Detailed Description

template<typename Service>
class asio::basic_deadline_timer< Service >

Provides waitable timer functionality.

The basic_deadline_timer class template provides the ability to perform a blocking or asynchronous wait for a timer to expire.

Most applications will use the asio::deadline_timer typedef.

Thread Safety:
Distinct objects: Safe.
Shared objects: Unsafe.
Concepts:
Async_Object.

template<typename Service>
int asio::basic_deadline_timer< Service >::cancel (   ) 
 

Cancel any asynchronous operations that are waiting on the timer.

This function forces the completion of any pending asynchronous wait operations against the timer. The handler for each cancelled operation will be invoked with the asio::error::operation_aborted error code.

Returns:
The number of asynchronous operations that were cancelled.

template<typename Service>
void asio::basic_deadline_timer< Service >::wait (   ) 
 

Perform a blocking wait on the timer.

This function is used to wait for the timer to expire. This function blocks and does not return until the timer has expired.

Exceptions:
asio::error Thrown on failure.

template<typename Service>
template<typename Handler>
void asio::basic_deadline_timer< Service >::async_wait (  Handler  handler  ) 
 

Start an asynchronous wait on the timer.

This function may be used to initiate an asynchronous wait against the timer. It always returns immediately, but the specified handler will be notified when the timer expires, or if the operation is cancelled.

Parameters:
handler The handler to be called when the timer expires. Copies will be made of the handler as required. The equivalent function signature of the handler must be:

   const asio::error& error // Result of operation
 );
