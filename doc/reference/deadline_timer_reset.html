Changing an active deadline_timer's expiry time

Changing the expiry time of a timer while there are asynchronous waits on it has undefined behaviour. To safely change a timer's expiry, pending asynchronous waits need to be cancelled first. This works as follows:

  • The asio::basic_deadline_timer::cancel() function returns the number of asynchronous waits that were cancelled. If it returns 0 then you were too late and the wait handler has already been executed, or will soon be executed. If it returns 1 then the wait handler was successfully cancelled.
  • If a wait handler is cancelled, the asio::error passed to it contains the value asio::error::operation_aborted.
For example, to reset a timer's expiry time in response to some event you would do something like this:

 void on_some_event()
 {
   if (my_timer.cancel() > 0)
   {
     // We managed to cancel the timer. Set new expiry time.
     my_timer.expires_from_now(seconds(5));
     my_timer.async_wait(on_timeout);
   }
   else
   {
     // Too late, timer has already expired!
   }
 }

 void on_timeout(const asio::error& e)
 {
   if (e != asio::error::operation_aborted)
   {
     // Timer was not cancelled, take necessary action.
   }
 }
